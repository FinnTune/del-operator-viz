# Del Operator Visualizer

A Jupyter Lab project for building intuition about the **del operator** (∇) through hand-derivation, symbolic verification, numerical computation, and visualization on real and synthetic data.

The goal is *not* to be a fast vector-calculus library (NumPy, SciPy, and SymPy already exist). The goal is to make every step inspectable, so you can connect the textbook formulas to working code and pictures.

## What's covered

| Notebook | Operator | What it does |
|---|---|---|
| `01_gradient.ipynb` | ∇f | scalar field → vector field (steepest ascent) |
| `02_divergence.ipynb` | ∇·F | vector field → scalar field (source/sink strength) |
| `03_curl.ipynb` | ∇×F | vector field → vector field (axis & rate of rotation) |
| `04_real_data.ipynb` | All three | applied to terrain elevation and 2D fluid flow |

Each notebook follows the same four-step structure: **derive by hand → verify with SymPy → compute numerically with NumPy → visualize with Matplotlib.**

## Mathematical foundation

The del operator in Cartesian ℝ³ is

$$\nabla = \left(\frac{\partial}{\partial x},\ \frac{\partial}{\partial y},\ \frac{\partial}{\partial z}\right)$$

It only has meaning once it acts on a field. The three fundamental actions are:

### 1. Gradient — `∇f` (scalar in, vector out)

$$\nabla f = \left(\frac{\partial f}{\partial x},\ \frac{\partial f}{\partial y},\ \frac{\partial f}{\partial z}\right)$$

Points in the direction of steepest ascent of `f`; magnitude is the rate of ascent.

**Worked example.** For `f(x, y) = x²y + sin(y)`:

- `∂f/∂x = 2xy` (treat y as constant)
- `∂f/∂y = x² + cos(y)` (treat x as constant)
- So `∇f = (2xy, x² + cos(y))`

At point `(1, 0)`: `∇f = (0, 2)` — the function climbs fastest in the +y direction.

### 2. Divergence — `∇·F` (vector in, scalar out)

For a vector field `F = (F_x, F_y, F_z)`:

$$\nabla \cdot \mathbf{F} = \frac{\partial F_x}{\partial x} + \frac{\partial F_y}{\partial y} + \frac{\partial F_z}{\partial z}$$

Measures the net "outflow density" at a point. Positive divergence = source (water spraying outward); negative = sink (water draining); zero = incompressible flow.

**Worked example.** For `F(x, y) = (xy, y² − x)`:

- `∂F_x/∂x = y`
- `∂F_y/∂y = 2y`
- So `∇·F = y + 2y = 3y`

Above the x-axis the field has a net source; below, a net sink.

### 3. Curl — `∇×F` (vector in, vector out)

In 3D:

$$\nabla \times \mathbf{F} = \left(\frac{\partial F_z}{\partial y} - \frac{\partial F_y}{\partial z},\ \frac{\partial F_x}{\partial z} - \frac{\partial F_z}{\partial x},\ \frac{\partial F_y}{\partial x} - \frac{\partial F_x}{\partial y}\right)$$

The curl vector points along the rotation axis (right-hand rule), with magnitude proportional to twice the local angular velocity.

**Worked example.** For the 2D rotation field `F(x, y, z) = (−y, x, 0)`:

- `∂F_z/∂y − ∂F_y/∂z = 0 − 0 = 0`
- `∂F_x/∂z − ∂F_z/∂x = 0 − 0 = 0`
- `∂F_y/∂x − ∂F_x/∂y = 1 − (−1) = 2`
- So `∇×F = (0, 0, 2)`

Constant rotation about +z, like a rigid disk spinning counter-clockwise.

In **2D**, curl reduces to a scalar: `(∇×F)_z = ∂F_y/∂x − ∂F_x/∂y`.

## Numerical recipe (the bridge from math to code)

Analytical derivatives need a formula for the field. On real data we only have values on a grid, so we approximate derivatives with **central finite differences**:

$$\frac{\partial f}{\partial x}\bigg|_{i,j} \approx \frac{f_{i+1,j} - f_{i-1,j}}{2 \Delta x}$$

This is `O(Δx²)` accurate (much better than forward differences, almost free to compute). At grid edges we fall back to one-sided differences. NumPy's `np.gradient` does exactly this.

From there:

- **Gradient** is two `np.gradient` calls (one per axis).
- **Divergence** is the sum of `np.gradient(F_x, axis=0) + np.gradient(F_y, axis=1)`.
- **Curl (2D)** is `np.gradient(F_y, axis=0) − np.gradient(F_x, axis=1)`.

The `src/operators.py` module wraps these so you can see exactly what's happening — no black box.

## Setup

Tested on Python 3.12+ (NumPy 2.5+). On your Arch box, `python` likely points at 3.14 — that's fine.

```bash
git clone https://github.com/FinnTune/del-operator-viz.git
cd del-operator-viz
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
jupyter lab
```

Then open the `notebooks/` folder and start with `01_gradient.ipynb`.

## Suggested learning path

1. Read this README's *Mathematical foundation* section.
2. Work `01_gradient.ipynb` cell by cell — try to predict each output before running.
3. Do the by-hand exercises at the bottom of each notebook before checking with SymPy.
4. Skip ahead to `04_real_data.ipynb` once all three operators feel concrete.
